# Introduction à l'API MDT

## Vue d'ensemble

Cette API constitue un pont de communication essentiel entre votre Mobile Data Terminal (MDT) et les serveurs FiveM utilisant les frameworks ESX ou QB-Core. Elle a été conçue pour automatiser et simplifier l'accès aux données des citoyens stockées dans la base de données de votre serveur de jeu de rôle.

## Objectif principal

L'objectif principal de cette API est de permettre la **récupération automatique et en temps réel des informations des citoyens** directement depuis la base de données du serveur FiveM. Cette intégration élimine le besoin de saisie manuelle des données et garantit que les informations affichées dans le MDT sont toujours à jour et synchronisées avec l'état actuel du serveur.

## Fonctionnalités clés

- **Récupération automatique des profils citoyens** : Accès instantané aux données personnelles, antécédents, et informations de base
- **Synchronisation en temps réel** : Les modifications apportées sur le serveur sont immédiatement reflétées dans le MDT
- **Compatibilité multi-framework** : Support natif pour les serveurs ESX et QB-Core
- **Interface unifiée** : Une seule API pour gérer les données, quel que soit le framework utilisé

## Avantages

Cette solution automatisée permet aux forces de l'ordre et aux équipes administratives de :

- Accéder rapidement aux informations des citoyens sans quitter l'interface MDT
- Réduire les erreurs liées à la saisie manuelle
- Améliorer l'efficacité des contrôles et des enquêtes
- Maintenir une cohérence des données entre le serveur de jeu et les outils externes

L'API est conçue pour être robuste, sécurisée et facilement intégrable dans votre écosystème existant.

## Architecture technique

Cette API est développée avec **NestJS**, un framework Node.js moderne et évolutif. Elle nécessite **Node.js version 22.x** ou supérieure pour fonctionner correctement.

## Installation

### Prérequis

- Node.js 22.x ou version supérieure
- npm ou yarn (gestionnaire de paquets)
- Git
- Accès à la base de données de votre serveur FiveM

### Méthode 1 : Installation via Coolify

L'API peut être déployée facilement via **Coolify**, une plateforme de déploiement qui simplifie la gestion des applications. Cette méthode est recommandée pour les déploiements en production et offre une interface de gestion intuitive.

### Méthode 2 : Installation manuelle

Pour une installation manuelle, suivez ces étapes :

```
# Cloner le repository
git clone https://github.com/urahost/uracad-fivem-api.git

# Accéder au répertoire du projet
cd uracad-fivem-api

# Installer les dépendances
npm install

# Construire l'application
npm run build

# Démarrer l'API
npm run start
```

Une fois l'installation terminée, l'API sera accessible et prête à être configurée pour se connecter à votre serveur FiveM.

## Configuration

### Variables d'environnement

Avant de démarrer l'API, vous devez configurer les variables d'environnement selon le type de serveur FiveM que vous utilisez. Créez un fichier `.env` à la racine du projet et ajoutez les variables appropriées :

#### Pour un serveur ESX

```
ESX_DATABASE_URL=mysql://username:password@host:port/database_name
```

#### Pour un serveur QB-Core

```
QBCORE_DATABASE_URL=mysql://username:password@host:port/database_name
```

#### Configuration mixte (optionnel)

Si vous gérez plusieurs serveurs avec des frameworks différents, vous pouvez configurer les deux variables :

```
ESX_DATABASE_URL=mysql://username:password@host:port/esx_database
QBCORE_DATABASE_URL=mysql://username:password@host:port/qbcore_database
```

**Important** : L'API détectera automatiquement le type de framework en fonction de la variable d'environnement configurée et adaptera ses requêtes en conséquence.

### Format de l'URL de base de données

Les URLs de base de données doivent respecter le format suivant :

- **Hôte** : Adresse IP ou nom de domaine du serveur de base de données
- **Port** : Port de connexion MySQL (généralement 3306)
- **Base de données** : Nom de la base de données de votre serveur FiveM
- **Identifiants** : Nom d'utilisateur et mot de passe avec les droits de lecture sur la base
